Cheat Sheet – Schnellübersicht

                
Zentrale Konzepte auf einen Blick. Ideal zum schnellen Nachschlagen vor der Prüfung.

            


            

                
                
                

                    

                         JavaScript: var / let / const
                    

                    

                        
var Function Scope • Hoisting • kann re-deklariert werden

                        
let Block Scope • keine Hoisting • kann reassigned werden

                        
const Block Scope • muss initialisiert werden • Referenz unveränderbar (nicht Inhalt)

                    

                    
Empfehlung: Immer let oder const verwenden. Nie implizit globale Variablen!

                


                
                

                    

                         REST & HTTP-Methoden
                    

                    

                        
GET — Read / Abrufen

                        
POST — Create / Erstellen

                        
PUT — Update (vollständig)

                        
DELETE — Delete / Löschen

                    

                    

                        REST-Prinzipien: Stateless, Uniform Interface, Ressourcen über URIs identifizierbar, Repräsentationen (JSON)
                    

                


                
                

                    

                         Web-Security
                    

                    

                        

                            
SQL-Injection (SQLi)

                            
Schädlicher SQL-Code in Formulare/URL-Parameter. Schutz: Prepared Statements / Parameterized Queries.

                        

                        

                            
Cross-Site Scripting (XSS)

                            
Böswilliges JS in Webseiten. Schutz: Content-Security-Policy (CSP), escaping.

                        

                        

                            
CSRF

                            
Gefälschte Requests mit gültiger Authentifizierung. Schutz: CSRF-Token + SameSite Cookies.

                        

                    

                


                
                

                    

                         Cookies & Sessions
                    

                    

                        
HTTP ist stateless. Cookies ermöglichen State.

                        
Server setzt Cookie via Set-Cookie Header.

                        
Browser sendet Cookie automatisch im Cookie Header zurück.

                        
                        

                            
HttpOnly — Kein JS-Zugriff

                            
Secure — Nur HTTPS

                            
SameSite — CSRF-Schutz

                            
Max-Age / Expires — Persistenz

                        

                    

                


                
                

                    

                         Event Loop & Async
                    

                    

                        Node.js & Browser sind single-threaded mit Event Loop.


                        Call Stack → Web APIs / libuv → Callback Queue → Event Loop


                        Callbacks → Promises → async/await für bessere Lesbarkeit.
